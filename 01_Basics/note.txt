基础语法
1.变量与数据类型
(1) What（是什么）
    变量：存储数据的命名容器，本质是内存地址的引用标签
    四大基础类型：
    int：整数，无大小限制（自动长整型）
    float：浮点数，带小数的数值（IEEE 754双精度）
    str：字符串，不可变的字符序列
    bool：布尔值，仅 True 和 False 两个值
    type()：内置函数，返回对象的类型信息
    强制类型转换：通过 int()、float()、str() 等函数将数据从一种类型转为另一种
(2) Why（为什么）
    变量：让数据具有可读性和复用性，避免硬编码和重复计算
    类型系统：明确数据性质，决定可执行的操作（如字符串不能乘字符串）
    type()：动态类型语言中，调试和类型检查的核心工具
    类型转换：实现不同数据类型的兼容运算（如数字拼接成字符串输出）
(3) How（怎么用）
    变量命名：遵循PEP8规范，小写字母+下划线，见名知意（如 user_age）
    类型选择：数值计算用 int/float，文本处理用 str，逻辑判断用 bool
    type() 使用：在交互环境或调试时验证变量类型，确保数据符合预期
    转换时机：输入数据默认是字符串，计算前需转数字；输出时需转字符串格式化

2. 输入与输出
(1) What（是什么）
    input()：从标准输入（键盘）读取用户输入，始终返回字符串
    print()：向标准输出（屏幕）打印内容，可输出多个对象，自动添加空格和换行
    f-string：格式化字符串字面值，以 f"..." 形式在字符串内嵌入变量和表达式
(2) Why（为什么）
    input()：实现程序与用户的动态交互，让程序行为由外部数据驱动
    print()：程序结果的展示窗口，调试时观察变量值的主要手段
    f-string：Python 3.6+最佳格式化方案，比 % 和 format() 更直观、高效、易读
(3) How（怎么用）
    input()：配合提示语 input("请输入：")，输入后务必根据需求进行类型转换
    print()：控制分隔符 sep=' ' 和结尾符 end='\n'，实现不换行或特殊格式输出
    f-string：在 {} 内直接写变量名或表达式，支持格式控制 {value:.2f} 保留两位小数
    核心原则：输入即字符串，输出即展示，f-string是连接二者的优雅桥梁

3. 运算符
What（是什么）
    算术运算符：+ 加、- 减、* 乘、/ 真除法、// 地板除（向下取整）、% 取模（求余）、** 幂运算
    比较运算符：> 大于、< 小于、== 等于、!= 不等于、>= 大于等于、<= 小于等于
    逻辑运算符：and 与（两者真才真）、or 或（一者真即真）、not 非（取反）
Why（为什么）
    算术：实现数学计算，是数据处理的基础工具
    比较：进行条件判断，决定程序流程走向
    逻辑：组合多个条件，表达复杂判断逻辑（如区间判断 0 < x < 10）
How（怎么用）
    优先级：** > 正负号 > * / // % > + - > 比较 > not > and > or
    类型影响：+ 对数字是加法，对字符串是拼接；* 对字符串是重复
    短路特性：and 和 or 遇到确定结果会停止计算（如 False and X 不计算X）
    链式比较：Python支持 a < b < c 这样简洁的数学式表达

4. 流程控制
(1) What（是什么）
    if/elif/else：条件分支结构，根据布尔表达式的真假选择执行路径
    while 循环：条件循环，当条件为真时重复执行代码块
    for 循环：迭代循环，遍历可迭代对象（列表、字符串、range等）的每个元素
(2) Why（为什么）
    if：让程序具备决策能力，实现“不同情况不同处理”
    while：适合未知次数的循环（如用户输入验证、游戏主循环）
    for：遍历数据集合的首选，结构清晰，不易死循环
(3) How（怎么用）
    if结构：if 开头，elif 可选（可多个），else 兜底。条件按顺序判断，命中即退出
    while风险：必须确保条件最终会变为False，否则陷入死循环
    for遍历对象：
    range(n)：生成 0 到 n-1 的整数序列，适合固定次数循环
    列表：直接获取元素值，无需手动索引
    字符串：逐个字符遍历，实现逐字符处理
    统一原则：所有流程控制语句后加冒号 :，代码块统一缩进（4个空格）

5. 容器（轻度）
(1) What（是什么）
    list（列表）：有序、可变、可重复的元素序列，用 [] 表示
    tuple（元组）：有序、不可变、可重复的元素序列，用 () 表示
    dict（字典）：无序（3.7+有序）的键值对集合，键唯一且不可变，用 {} 表示
    set（集合）：无序、不重复的元素集合，用 {} 或 set() 表示
    索引：通过整数下标访问单个元素（list[0]）
    切片：通过范围提取子序列（list[1:3]）
    常用方法：
    append：列表末尾添加元素
    pop：移除并返回指定位置元素
    len：返回容器元素个数
    in：判断元素是否存在于容器中
(2) Why（为什么）
    列表：最常用的数据容器，动态数组，支持增删改查
    元组：不可变特性使其适合作为字典键、函数返回值、保护数据不被修改
    字典：通过键快速查找值（O(1)），是映射关系的最佳实现
    集合：数学集合运算（交并差），高效去重和成员判断
    索引/切片：快速访问数据子集，是数据提取的核心操作
    常用方法：提供高频操作的现成接口，避免手动实现
(3)How（怎么用）
    列表：存储同类型数据序列，需要修改时用
    元组：存储异构数据（如坐标 (x, y)），不需要修改时用
    字典：存键值映射，通过键名快速访问值（如 user_info['name']）
    集合：快速去重（set(lst)）、判断交集并集
    索引：正向从0开始，负索引从-1（末尾）开始
    切片：[start:stop:step]，可省略参数，[:] 复制容器
    方法选择：append 追加，pop 删除并获取（栈结构），len 获取长度（通用函数），in 做成员判断（比循环高效）

6. PEP8代码风格
(1）What（是什么）
    变量命名：小写字母 + 下划线分隔（如 user_name）
    行宽限制：每行不超过79字符，长表达式可换行或用小括号包裹
    运算符空格：二元运算符两侧各加一个空格（如 a + b）
    函数间距：函数定义之间空一行，类的方法之间空一行
（2）Why（为什么）
    团队协作：统一风格让代码易于阅读和维护，降低沟通成本
    可读性：空格和换行让代码结构清晰，减少视觉疲劳
    专业性：遵守社区规范是专业开发者的基本素养，提升代码质量
    工具支持：遵循PEP8可无缝使用 flake8、black 等自动化工具
（3）How（怎么用）
    命名：函数和变量用 snake_case，类名用 PascalCase，常量用 UPPER_CASE
    行宽：在IDE中设置竖线提示（79字符），过长时用小括号 () 隐式续行
    空格：+ - * / == 等两侧都加空格，冒号 : 和逗号 , 后加空格（前不加）
    空行：顶级函数/类之间空两行，类内方法之间空一行，逻辑段落间用空行分隔
    养成习惯：安装编辑器插件（如Python-autopep8），保存时自动格式化
