模块与包 + 实战脚本 整体介绍
(1) What（是什么）
    这是 Python 工程化的核心能力，将代码从"能跑"提升到"可维护、可复用、可交付"。它包含五大支柱：
        import 机制：复用代码的"导入系统"
        自定义模块：自己编写 .py 文件作为工具库
        标准库模块：random、os、sys 等开箱即用的高频工具
        脚本化思维：将程序抽象为"输入 → 处理 → 输出"的流水线
        命令行脚本：独立可执行工具，脱离 IDE 在终端直接运行
    关系：import 是语法，模块是载体，标准库是武器，脚本化思维是设计模式，最终产出是生产工具。

(2) Why（为什么必须掌握）
    不会模块化的灾难：
        所有代码挤在一个文件，几千行后无法维护
        重复造轮子，每个项目重写一遍文件遍历、随机数生成
        脚本只能在 PyCharm 点运行，无法集成到自动化流程
    掌握后的价值：
        代码复用：一次编写，到处 import，团队共享
        标准工具：os 处理文件路径，sys 读取命令行参数，无需自己解析
        自动化能力：命令行脚本是 CI/CD、定时任务、运维工具链的基本单元
        工程思维："输入→处理→输出"是所有程序的元模型，掌握后能快速拆解任何需求
        面试价值：能独立写出命令行脚本，证明你不是只会做练习题，而是能交付生产工具。

(3) How（整体学习路径）
    步骤 1：理解 import 三种语法差异，能自如导入标准库和自定义模块
    步骤 2：动手写一个 utils.py，封装 3 个常用函数，体验模块诞生
    步骤 3：用 random 生成测试数据，os 遍历目录，sys 读取命令行参数，完成一个"批处理图片重命名"脚本
    步骤 4：用"输入→处理→输出"框架重构脚本，确保可测试
    步骤 5：在终端执行 python mytool.py ./images --prefix=img_，交付第一个命令行工具
